# NAS Model Root Layout Plan

This document is a **plan only**. Do not move NAS model folders from GitHub. The
current runtime body is still `X:\Backside\_models\_Models`; the cleaner target
front door is `X:\Models` or a `MODELS_ROOT` alias that resolves there after an
approved NAS migration.

## Doctrine

- GitHub is the governor/spec layer.
- NAS is the runtime/body layer.
- Do not delete anything from the model root in this planning PR.
- Do not upload model weights, caches, virtualenvs, generated reports, logs,
  databases, vector indexes, or secrets.
- Prefer `%MODELS_ROOT%` / `$env:MODELS_ROOT` indirection over hardcoded paths.

## Current problem

`X:\Backside\_models\_Models` mixes canonical model slots with fallbacks, Hugging
Face caches, virtualenvs, scripts, health reports, legacy folders, and loose ops
files. That makes the NAS body harder to reason about than the GitHub registry.

## Disposition buckets

| Bucket | Meaning |
| --- | --- |
| `KEEP_AS_MODEL_SLOT` | Canonical M01-M12 / P01-P07 slot folders. |
| `MOVE_TO_FALLBACKS` | Safe temporary fallbacks for degraded slots. |
| `MOVE_TO_CACHE` | Hugging Face or package caches, not canonical slots. |
| `MOVE_TO_ENVS` | Virtualenvs or local environment folders. |
| `MOVE_TO_OPS` | Scripts, health reports, batch files, requirements, and operational docs. |
| `MOVE_TO_LEGACY` | Historical copies kept for rollback/reference. |
| `NEEDS_DAVID_CALL` | Anything not clearly classifiable before move time. |

## Migration packet

`models/MODEL_ROOT_MIGRATION_PACKET.json` is generated by
`scripts/generate_model_root_migration_packet.py`. It records the current root,
target root, hardcoded reference audit, disposition plan, risky moves avoided,
and next PR plan.

## Runtime migration sequence for a later PR

1. Confirm or create `$env:MODELS_ROOT` / `%MODELS_ROOT%` on NAS.
2. Generate a dry-run move report from the packet; do not move unknown files.
3. After approval, create `X:\Models` target buckets.
4. Move slots and support folders in controlled batches.
5. Update `models/MODEL_REGISTRY.json`, `models/MODEL_FALLBACKS.json`, runtime
   scripts, station configs, and docs from old hardcoded root to `MODELS_ROOT` or
   `X:\Models`.
6. Run canaries and model health checks before retiring fallback aliases.
